# Marc 用户常见痛点

这个项目面向已经在用 MSC Marc/Mentat 的用户，尤其是经常被流程细节卡住的人。

## 常见问题

- Mentat 操作重复，模型调整过程很难追踪。
- Codex/AI 很聪明，但不能稳定、受控地连接 Marc。
- 求解失败后，`.out/.sts`、模型设置、边界条件分散在不同地方。
- 节点集、单元集、RBE/link、载荷边界需要反复检查。
- `.t16` 结果提取、曲线整理、CSV/图片输出很耗时间。
- 论文或项目需要留下清晰证据：改了什么、查了什么、结果来自哪里。

## Marc MCP Assistant 想解决什么

它不是替代 Marc，也不是替代工程判断。

它的目标是把 Marc/Mentat 的重复操作、诊断线索和结果整理，接到一个受控的 AI 辅助工作流里：

```text
本地 Marc/Mentat
-> MCP 工具层
-> AI 辅助建模/检查/诊断/后处理
-> 项目本地证据和结果
```

## 适合谁试用

- 正在用 Marc 做论文、科研或工程模型；
- 经常需要反复改模型；
- 遇到求解错误、收敛问题、结果提取问题；
- 希望用 Codex/AI 辅助 Marc 工作流；
- 愿意先从小范围试用开始验证。

## 不适合谁

- 没有本地 Marc/Mentat 安装和授权；
- 希望工具自动保证模型正确；
- 希望公开获取全部源码；
- 希望免费获得无限制项目调试。
