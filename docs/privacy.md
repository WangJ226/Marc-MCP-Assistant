# 隐私说明

## 基本原则

模型属于用户。工具不应该偷偷收集、上传或复用用户模型。

## 默认行为

默认情况下，诊断和试用流程应尽量在本地完成，不主动上传：

- `.mud`
- `.mfd`
- `.dat`
- `.t16`
- `.t19`
- 完整项目文件夹

## 可选分享

如果需要支持，用户可以自愿分享诊断信息。建议优先分享：

- Marc 版本；
- 工具版本；
- 错误码或 solver exit number；
- 选定的 `.out/.sts` 摘录；
- 脱敏日志摘要；
- 用户自己选择的截图。

## 敏感信息

发送任何资料前，建议删除或遮挡：

- 项目名称；
- 公司名称；
- 完整本地路径；
- 保密几何；
- 敏感材料参数；
- license server 信息。

## 案例贡献

用户可以自愿贡献脱敏案例，用于改进后续诊断规则和产品能力。贡献应当是明确、自愿、可确认的。
