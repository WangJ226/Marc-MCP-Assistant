# 试用流程

## 当前阶段

当前是 Windows 闭源灰度试用阶段。包不会公开源码，试用需要本地 license。

早期流程：

```text
联系作者
-> 说明 Marc 版本、Windows 版本和当前问题
-> 作者判断是否适合当前试用包
-> 获取闭源包和本地 license
-> 本机运行检查
-> 试用有限工作流
-> 反馈问题或转为付费使用
```

## 后续公开下载计划

闭源包经过真实用户测试后，后续可能放到 GitHub Releases。届时用户可以：

```text
从 GitHub Releases 下载 Windows 闭源包
-> 运行机器码/状态检查
-> 联系作者申请试用 license
-> 导入 license 后试用
```

公开下载包本身不等于可无限使用。没有有效 license，核心功能不会开放。

## 用户需要准备

- Windows 电脑；
- 已能正常启动的本地 MSC Marc/Mentat；
- 有效 Marc/Mentat 授权；
- 一个备份后的测试模型或公开示例模型；
- 当前最想解决的问题描述。

## 试用可能限制

- 有效时间；
- 绑定机器；
- 运行次数；
- 功能组；
- 支持的 Marc 版本；
- 高级诊断、批处理和后处理能力。

## 反馈方式

试用过程中如果出现问题，优先发送：

- 错误截图；
- 日志摘录；
- Marc 版本和 Windows 版本；
- 你执行到哪一步；
- 是否可以分享脱敏后的输入或结果片段。

不默认收集完整模型文件。需要完整模型诊断时，会单独确认。

## 付费方式

早期付费方式会根据场景人工确认，可能包括：

- 单次问题诊断；
- 短期试用延长；
- 月度使用；
- 项目级支持；
- 定制工作流整理。

具体价格和范围会随版本成熟度调整。
