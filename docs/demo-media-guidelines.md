# Demo Media Guidelines

Public screenshots and videos should prove value without exposing private
implementation details.

## Good Demo Materials

- Short screen recording of a safe demo workflow.
- Screenshot of a generated diagnosis report using synthetic or anonymized data.
- Plot or curve exported from a demo result.
- High-level before/after workflow diagram.
- Redacted terminal or Codex interaction showing product-level behavior only.

## Do Not Show

- Source code.
- Runtime scripts or MCP configuration.
- Local absolute paths.
- License server or license files.
- Real project names, company names, or user names.
- Full Marc model files or confidential geometry.
- Private case identifiers.
- Internal package names that reveal implementation structure.

## Redaction Checklist

Before publishing any image or video, check:

- model name is synthetic or anonymized;
- local path is hidden or blurred;
- license information is hidden;
- user/computer name is hidden;
- no private file tree is visible;
- no internal source file is visible;
- no real model file is downloadable from the public repository.

## Suggested First Demo Set

For the first public showcase, prepare:

- one workflow screenshot;
- one diagnosis-report screenshot;
- one result-curve image;
- one 30-60 second short video with sensitive areas blurred.
