# 第一位灰度用户测试清单

这份清单用于闭源包公开放到 GitHub Releases 之前。建议先找一位可信 Marc 用户完整走一遍真实流程。

## 测试目标

确认普通用户能否完成：

- 读懂公开页；
- 联系作者申请试用；
- 下载或接收闭源包；
- 获取机器信息；
- 导入本地 license；
- 运行状态检查；
- 连接 Codex/AI 工作流；
- 完成一个有限 Marc/Mentat 测试任务；
- 反馈问题。

## 测试前准备

测试用户需要：

- Windows 电脑；
- 本地可正常启动的 MSC Marc/Mentat；
- 有效 Marc/Mentat 授权；
- 一个可公开或可脱敏的测试模型；
- 愿意反馈安装和使用过程。

## 建议测试步骤

```text
1. 用户只看 GitHub 页面，不看你的内部说明。
2. 用户按页面提示联系你。
3. 你按真实流程让用户提供 Marc 版本、Windows 版本和问题描述。
4. 你发送闭源包和初始说明。
5. 用户解压并运行状态检查。
6. 用户把机器信息发给你。
7. 你生成并发送试用 license。
8. 用户导入 license 并再次检查。
9. 用户完成一个小型 Marc/Mentat 工作流。
10. 用户按反馈模板发回问题。
```

## 必测问题

- 没有 license 时，核心功能是否无法使用？
- license 是否能在目标电脑生效？
- license 是否只对预期版本和机器生效？
- 用户是否能看懂安装说明？
- 用户是否知道不要一开始发送完整模型？
- 错误信息是否足够让你判断问题？
- 包内是否没有源码、私钥、内部路径和客户资料？

```text
卡在哪一步：

最不清楚的地方：
最有用的功能：
愿意付费的使用方式：单次/按月/项目支持/暂不确定
```

## 通过标准

第一位灰度用户不需要完全没有问题。只要满足下面条件，就可以考虑准备 GitHub Release：

- 授权链路能跑通；
- 用户不需要你远程操作也能完成基本检查；
- 核心功能没有明显绕过授权；
- 反馈能被记录和转化为下一版改进；
- 公开页没有造成明显误解。
